Wenn man ein neues Programm programmieren will, sollte mal normalerweise erst mal mit einer ausführlichen Planung beginnen, was sehr wichtig für ein gutes, sauberes und stabiles Programm ist. Bei diesem Programm ist das aber jetzt nicht nötig. Danach werden dann erstmal alle Steuerelemente auf der Form platziert und ihre Eigenschaften definiert. Setzen Sie jetzt mal 2 Buttons auf die Form. Danach ändern Sie erstmal ihre Namen (das ist die Eigenschaft "(Name)"; unter diesem Namen werden diese Steuerelemente später im Code angesprochen). Nennen Sie den einen Button "KlickButton" und den anderen "EndeButton". Danach können Sie unter der Eigenschaft "Caption" die Aufschrift für diese Buttons festlegen. Schreiben Sie dort etwas entsprechendes. Das ganze sollte dann in etwa so aussehen:

Nach einem Doppelklick auf eines der platzierten Objekte gelangen Sie immer sofort in den Code zu dem Standard-Ereignis des entsprechenden Objektes, in diesem Fall ist das das Click-Ereignis, welches ausgeführt wird, wenn man auf das Button klickt. Schreiben Sie jetzt bei dem KlickButton folgenden Code:
MsgBox "Hallo Welt!"
Beim EndeButton schreiben Sie folgenden Code:
End

Wenn Sie das Programm jetzt starten, öffnet sich das von Ihnen bearbeitete Fenster. Wenn Sie auf den KlickButton klicken, dann kommt folgende Meldung:

Nach einem Klick auf den EndeButton wird das Programm beendet.

Das ganze läßt sich natürlich jetzt problemlos erweitern. Gucken Sie einfach mal die Eigenschaften durch und probieren Sie einfach mal ein bisschen. Das kann nie schaden - im Gegenteil, Sie lernen dabei sogar neues! Sie können natürlich auch versuchen, denn Code beim KlickButton zu Erweitern. Fügen Sie mal ein Komma hinter dem MsgBox-Befehl ein. Visual Basic müsste jetzt eigentlich die möglichen Parameter anzeigen. Hier bei MsgBox sind alle weiteren Befehle optional, dass heißt, sie müssen nicht angegeben werden. Wenn sie nicht angegeben werden, wird enfach der Standard-Wert genommen. Daher kommt beispielsweise auch die unschöne Überschrift dieser Meldung. Versuchen Sie selbst einmal rauszufinden, wie Sie jetzt den Titel ändern können. Außerdem können Sie dann auch mal andere Steuerelemente testen. Setzen Sie einfach mal alle vorhandenen Steuerelemente auf die Form, um zu sehen, was sich dahinter verbirgt - das meiste sollte Ihnen aus dem Windows-Alltag bekannt vorkommen.

Ich hoffe, Ihnen einen ersten Einblick in die Welt von Visual Basic geboten zu haben.
